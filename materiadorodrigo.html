🌐 Códigos HTTP

  • 200 OK → Requisição bem-sucedida
  • 201 Created → Novo recurso criado
  • 400 Bad Request → Requisição inválida
  • 401 Unauthorized → Usuário não autenticado
  • 404 Not Found → Recurso não encontrado
  • 500 Internal Server Error → Erro interno do servidor

🔒 HTTPS

A porta padrão do HTTPS é 443.

Método Função

⚙️ O que é REST?

REST (Representational State Transfer) é um conjunto de regras e diretrizes arquiteturais usado para que diferentes sistemas se comuniquem pela internet utilizando HTTP.

Regras do REST

  • Client-Server
  • Stateless (Sem estado)
  • Cacheable

🔄 REST x HTTP

HTTP é o protocolo responsável pelo transporte dos dados. REST define o padrão de organização desse transporte.

HTTP transporta e REST organiza.

🧩 O que são APIs?

APIs (Application Programming Interface) são conjuntos de regras que permitem que diferentes sistemas conversem entre si.

Como funciona?

O cliente envia uma requisição → o servidor processa → uma resposta é retornada.

Onde usamos APIs?

  • Login com Google
  • Aplicativos bancários
  • Mapas e geolocalização
  • Previsão do tempo
  • Redes sociais

Tipos de API

  • Públicas
  • Privadas
  • Parceiras
  • Internas

🛡️ WAF

WAF (Web Application Firewall) protege aplicações web e APIs.

Ele monitora e filtra tráfego HTTP/HTTPS, bloqueando ameaças como:

  • SQL Injection
  • XSS (Cross-Site Scripting)
  • Bots maliciosos

🏗️ Arquiteturas de API

  • REST
  • SOAP
  • gRPC
  • GraphQL

🔑 JWT (JSON Web Token)

JWT é um padrão para criação de tokens com assinatura digital.

O payload contém informações (claims) sobre o usuário ou sistema.

🚀 Métodos Ágeis

Conjunto de práticas para desenvolvimento rápido e iterativo.

Valores

  • Colaboração com o cliente
  • Resposta rápida a mudanças
  • Software funcionando acima de documentação extensa

Exemplos

  • Scrum
  • XP
  • Kanban

📋 Scrum

  • Product Owner → Define o backlog
  • Sprint Planning → Planejamento da Sprint
  • Daily Scrum → Reunião diária de 15 minutos
  • Sprint Review → Demonstração das funcionalidades
  • Sprint Retrospective → Melhoria contínua

💻 XP (Extreme Programming)

  • TDD (Test-Driven Development)
  • Pair Programming
  • Refatoração constante
  • Integração contínua
  • Feedback frequente

📌 Kanban

  • A Fazer
  • Em Progresso
  • Concluído

O Kanban utiliza modelo puxado e limites de WIP.

🧠 Orientação a Objetos

  • Abstração
  • Encapsulamento
  • Herança
  • Polimorfismo

🤝 OO + Métodos Ágeis

  • Facilita refatoração
  • Melhora testabilidade
  • Permite modularidade
  • Reutilização de código

🧪 Princípio F.I.R.S.T.

  • Fast → Testes rápidos
  • Independent → Testes independentes
  • Repeatable → Resultados repetíveis
  • Self-validating → Auto validação
  • Thorough → Cobrir casos extremos
